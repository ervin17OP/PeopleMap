## Operational workflow

- Jira is the task system of record
- GitHub hosts the code and branches
- OpenClaw can be driven from Telegram or the Control UI
- Codex and OpenClaw share the same local project workspace

## Ticket-driven development

- Read or create a Jira ticket first
- Create a branch with the Jira key
- Implement the work
- Commit with the Jira key
- Push to GitHub
- Comment back on the Jira ticket
